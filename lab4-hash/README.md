# OpenHashTable & ClosedHashTable

## Опис
Обидва методи застосовуються для розв’язання колізій у хеш-таблицях.

- **Open Hashing (відкрите хешування)** — зберігає елементи у **ланцюжках (списках)** при кожній комірці.  
- **Closed Hashing (закрите хешування)** — розміщує всі елементи **всередині таблиці**, шукаючи наступну вільну позицію при колізії.

## Характеристики
| Метод | Складність (середня) | Пам’ять | Стабільність | Особливість |
|--------|-----------------------|----------|----------------|--------------|
| Open   | O(1)                 | Вища (через списки) | ✅ Так | Легке розширення |
| Closed | O(1)                 | Менша | ❌ Ні | Без додаткових структур |

## Висновок
OpenHashTable краще для великих наборів даних і частих вставок,  
ClosedHashTable — для компактних таблиць із контрольованим навантаженням.
